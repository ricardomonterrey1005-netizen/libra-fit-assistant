# MICRONUTRIENTS — Scientific Research for FitRicardo

Scope: actionable rules for a micronutrient tracker, deficiency detector, and
alert system, grounded in NIH Office of Dietary Supplements (ODS), WHO, the
Dietary Reference Intakes (DRIs) from the U.S. National Academies (formerly
IOM), EFSA Dietary Reference Values, and peer-reviewed literature (PubMed).
All RDA/AI/UL figures below are adult values unless stated otherwise. Full
source list at the end.

---

## 1. Executive Summary — Top 10 Micronutrients to Prioritize

The nutrients below are the highest-leverage to track because they are (a)
commonly under-consumed in multiple populations, (b) have clear clinical
consequences when deficient, and (c) are detectable from a reasonably
complete meal log. Ordered by practical importance in a general fitness app:

| # | Nutrient | Why it matters |
|---|----------|---------------|
| 1 | Vitamin D (cholecalciferol) | Bone health, immune, muscle function; low sun + indoor lifestyles → global deficiency (NIH ODS). |
| 2 | Iron (heme + non-heme) | #1 nutritional deficiency worldwide (WHO). Critical for menstruating women, endurance athletes, vegans. |
| 3 | Vitamin B12 (cobalamin) | Nerve + red blood cell function; near-zero in plant foods → vegans require tracking/supplementation. |
| 4 | Calcium | Bone density, muscle contraction; commonly low in no-dairy diets. |
| 5 | Magnesium | Energy metabolism, sleep, muscle function; ~50% of US adults below EAR (NHANES). |
| 6 | Potassium | Cardiovascular; low in Western/processed diets (AI 3400/2600 mg rarely met). |
| 7 | Fiber | Glycemic control, microbiome, satiety; <5% of adults meet AI (AHA). |
| 8 | Omega-3 (EPA+DHA) | Cardiovascular + anti-inflammatory; low in non-fish-eating populations. |
| 9 | Folate (B9) | DNA synthesis, critical pre-conception/pregnancy. |
| 10 | Zinc | Immune, testosterone, wound healing; low bioavailability in high-phytate (vegan) diets. |

Secondary (track but lower alert priority): Vitamin A, Vitamin C, Vitamin K,
Vitamin E, Iodine, Selenium, Choline, B6, Thiamine, Riboflavin, Niacin.

## 4. Goal-Specific Priorities

Encode these as "tracking emphasis tags" per user profile.

**Fat loss (caloric deficit):**
- Priority: protein (1.6-2.2 g/kg), fiber (volume/satiety), D, magnesium,
  potassium, iron (esp. women), B-complex.
- Alert rule: if deficit >25% TDEE for ≥3 days, raise calorie floor warning.
- Alert rule: if deficit active AND protein <1.4 g/kg, flag muscle-loss risk.

**Muscle gain:**
- Priority: protein (1.6-2.2 g/kg), zinc, magnesium, iron, vitamin D.
- Creatine monohydrate 3-5 g/day (not a micronutrient but evidence-backed).

**Endurance (running/cycling):**
- Priority: iron (ferritin monitoring for females), sodium + potassium (sweat
  losses 500-1500 mg/L Na), B1/B2/B3 (energy metabolism scale with kcal),
  vitamin E (oxidative stress), carbohydrate adequacy.
- Alert rule: if weekly running volume >40 km AND iron <80% RDA → flag.

**Pregnancy / Lactation:**
- Priority: folate (600 mcg, ideally 400 mcg supplemental pre-conception),
  iron (27 mg), calcium (1000 mg), DHA (200-300 mg), iodine (220 mcg),
  choline (450 mg).

**Men >50:** D, B12 (absorption drops), zinc, magnesium, fiber.
**Women premenopause:** iron, folate, calcium, D.
**Women postmenopause:** calcium 1200, D 800 IU, B12, protein.
**Children/adolescents:** calcium, iron, D, zinc (not the primary user base
but note if a parent profile is added).

## 5. Detection Algorithm

### 5.1 Data model requirements
Per food in `nutritionDB.js`, store per-100g values (see §9). Per meal log
entry, store `foodId`, `grams`, `timestamp`.

### 5.2 Pseudocode

```js
// --- Constants -------------------------------------------------------------
const RDA = {
  // Keyed by profile bucket -> nutrient -> value (unit implied by nutrient)
  // Bucket example: "M_19_50", "F_19_50", "F_PREG", "M_51p"
  M_19_50: { vitamin_c:90, iron:8,  calcium:1000, b12:2.4, d:15, fiber:38, ... },
  F_19_50: { vitamin_c:75, iron:18, calcium:1000, b12:2.4, d:15, fiber:25, ... },
  // ...
};

const UL = {
  vitamin_c:2000, iron:45, calcium:2500, b6:100, niacin:35,
  vitamin_a:3000, vitamin_d:100, vitamin_e:1000, zinc:40,
  selenium:400, sodium:2300 /* CDRR */, magnesium_suppl:350, folate_syn:1000
};

// Bioavailability multipliers applied at intake-sum time
const BIOAVAIL = {
  iron_heme: 1.0,        // reference
  iron_nonheme: 0.4,     // ~40% of heme absorption on average mixed diet
  b12_plant: 0.0,        // only reliable from animal/fortified/supplement
  calcium_oxalate: 0.5,  // spinach etc. half-counted
};

// --- Profile bucketing ----------------------------------------------------
function bucket(profile) {
  if (profile.pregnant) return "F_PREG";
  if (profile.lactating) return "F_LACT";
  const s = profile.sex === "M" ? "M" : "F";
  const a = profile.age;
  if (a < 19) return `${s}_TEEN`;
  if (a <= 50) return `${s}_19_50`;
  if (a <= 70) return `${s}_51_70`;
  return `${s}_71p`;
}

// --- Core algorithm -------------------------------------------------------
function detectDeficiencies(mealLog, profile, windowDays = 7) {
  const rda = RDA[bucket(profile)];
  const days = groupByDay(mealLog).slice(-windowDays);   // last N days
  if (days.length === 0) return [];

  // 1. Sum daily intake per nutrient, applying bioavailability
  const totals = {};   // { nutrient: [day1, day2, ...] }
  for (const day of days) {
    const daySum = {};
    for (const entry of day.entries) {
      const food = nutritionDB[entry.foodId];
      const factor = entry.grams / 100;
      for (const n of Object.keys(food.nutrients)) {
        const raw = food.nutrients[n] * factor;
        const adj = applyBioavailability(n, raw, food);
        daySum[n] = (daySum[n] || 0) + adj;
      }
    }
    for (const n of Object.keys(daySum)) {
      (totals[n] ||= []).push(daySum[n]);
    }
  }

  // 2. Evaluate vs RDA & UL
  const findings = [];
  for (const nutrient of Object.keys(rda)) {
    const series = totals[nutrient] || [];
    const avg = series.length ? mean(series) : 0;
    const target = rda[nutrient];
    const pct = target ? (avg / target) * 100 : null;
    const daysBelow70 = series.filter(v => v < 0.7 * target).length;
    const daysAboveUL = UL[nutrient]
      ? series.filter(v => v > UL[nutrient]).length
      : 0;

    let severity = "ok";
    if (daysAboveUL >= 2)         severity = "excess";
    else if (avg < 0.5 * target)  severity = "severe_deficit";
    else if (daysBelow70 >= Math.ceil(windowDays * 0.6))
                                  severity = "chronic_deficit";
    else if (pct < 70)            severity = "low";
    else if (pct < 90)            severity = "borderline";

    if (severity !== "ok") {
      findings.push({
        nutrient,
        currentAvg: round(avg, 2),
        rda: target,
        pctRda: round(pct, 0),
        daysBelow70,
        severity,
        suggestions: suggestFoods(nutrient, profile, 3),
        supplementAdvice: shouldSuggestSupplement(nutrient, profile, series)
      });
    }
  }
  // Rank by severity then by clinical priority from §1
  return findings.sort(byPriority);
}

// --- Alerting policy -------------------------------------------------------
// Run detectDeficiencies nightly (or on meal-log save).
// - severe_deficit OR chronic_deficit -> push notification + in-app banner
// - low -> weekly digest entry
// - borderline -> silent, shown only in "Nutrition insights" tab
// - excess -> immediate warning, especially for iron, A, D, zinc, sodium
```

### 5.3 Minimum data window
- 3 days: enough for macros and high-frequency micros (Na, K, fiber, C).
- 7 days: needed for D, B12, omega-3 (episodic foods like fish/eggs).
- 14-30 days: use rolling average for fat-soluble vitamins (A, D, E, K) and
  B12 — body stores smooth daily variation.

## 7. Supplementation Decision Tree

Principle: food-first. Recommend supplementation only when dietary correction
is impractical or clinically indicated.

```
For each flagged deficiency:
  1. Is this a nutrient where supplementation is well-evidenced?
     - B12: YES if vegan, age >50, atrophic gastritis, metformin user.
             Dose: 250-500 mcg/day cyanocobalamin or 1000 mcg 2x/week.
     - Vitamin D: YES if serum 25(OH)D <50 nmol/L or high-latitude/indoor
             lifestyle. Dose: 600-2000 IU/day. Do not exceed 4000 IU without
             labs.
     - Iron: ONLY if ferritin <30 ng/mL (labs needed). Risk of overload,
             constipation. Never suggest blanket iron.
     - Omega-3 (EPA+DHA): YES if <1-2 fish servings/week. Dose: 250-500
             mg/day combined EPA+DHA. Vegan source: algae oil.
     - Calcium: preferred from food. Supplement only if intake <600 mg and
             diet correction fails; max 500 mg/dose; avoid with iron supps.
     - Folate: pre-conception/1st trimester: 400 mcg/day synthetic.
     - Iodine: pregnancy 150 mcg/day; otherwise iodized salt is enough.
     - Magnesium: glycinate/citrate 200-400 mg is safe for sleep/cramps;
             avoid oxide (low BA, laxative).

  2. Is user in a special population with mandatory supplementation?
     - Pregnant: prenatal (folate+iron+iodine+DHA).
     - Breastfed infant: vitamin D 400 IU (not app scope but parent profile).
     - Bariatric surgery, IBD, celiac: refer to clinician.

  3. Contraindications to surface in-app:
     - Warfarin users: warn before vitamin K changes, omega-3 high-dose.
     - Hemochromatosis / family history: never push iron.
     - Kidney disease: limit K, Mg, P, protein — require medical flag.
     - Thyroid disease: caution with kelp/iodine spikes.
     - Smokers: DO NOT recommend high-dose beta-carotene (CARET trial
       showed increased lung cancer).

  4. Always: "consult a healthcare professional before starting supplements"
     disclaimer, especially for pregnancy, chronic disease, medications.
```

Implementation: each nutrient in DB gets a `supplementPolicy` field with
values `{ recommend: "food_first" | "common" | "clinical_only" | "avoid",
maxSafeDose, notes }`.

## 8. Integration with Caloric-Deficit Tracking

### 8.1 Formulas

```js
// BMR — Mifflin-St Jeor (preferred for general population)
function bmr(kg, cm, age, sex) {
  const base = 10*kg + 6.25*cm - 5*age;
  return sex === "M" ? base + 5 : base - 161;
}

// TDEE
const ACTIVITY = {
  sedentary: 1.2,       // desk, no exercise
  light:     1.375,     // light exercise 1-3 d/wk
  moderate:  1.55,      // moderate 3-5 d/wk
  active:    1.725,     // hard 6-7 d/wk
  athlete:   1.9        // 2-a-days, physical job
};
const tdee = bmr(...) * ACTIVITY[profile.activity];

// Target calories for fat loss: 10-25% deficit
const deficitPct = clamp(profile.deficitPct ?? 0.20, 0.10, 0.25);
const targetKcal = Math.max(tdee * (1 - deficitPct), floor(profile));

// Hard floor to prevent eating disorders / severe restriction
function floor(p) {
  return p.sex === "M" ? 1500 : 1200;
}
```

### 8.2 Alerts

| Condition | Alert level | Message (EN) |
|-----------|-------------|--------------|
| 7-day avg deficit >25% of TDEE | WARN | "Your deficit is aggressive. Long-term risk: muscle loss, metabolic adaptation. Consider raising target by X kcal." |
| 3-day avg deficit AND protein <1.4 g/kg | WARN | "You're in a deficit with low protein — this increases muscle loss risk. Target: {1.6 × kg} g/day." |
| Kcal <floor for 2 days | HARD_STOP | "Intake below safe minimum. Add a meal or review logging." |
| Weight loss >1% bodyweight/week for 3 weeks | WARN | "Rapid loss. Slow to ~0.5-0.7% per week for sustainability." |
| Kcal surplus >20% for 7 days on "cut" goal | WARN | "Intake exceeds target. Adjust portions or reassess goal." |
| Fiber <50% AI for 5 days | INFO | "Low fiber — add legumes, fruits, vegetables." |
| Sodium >2300 mg for 3 days | INFO | "High sodium — CDRR 2300 mg." |

### 8.3 Weekly energy-balance view
Compute 7-day rolling `sum(kcal_in) - sum(tdee)` and show as a trendline.
Better than daily view because social/weekend meals swing daily numbers.

### Minimum viable set (11 fields) for v1 micronutrient tracking
`kcal, protein_g, carbs_g, fat_g, fiber_g, sodium_mg, potassium_mg, calcium_mg, iron_mg, vitamin_c_mg, vitamin_b12_mcg`.

This covers >80% of alertable deficiencies in typical users while keeping
data-entry load manageable. Add Priority 2 in v2; Priority 3 only if you
source a complete table (USDA FDC or INCAP dump).

## 10. "Quick-Add" Scenarios

| Scenario | UX pattern | Data policy |
|----------|-----------|-------------|
| Sudden meal (social/unexpected) | Floating "+ Add now" button always visible; defaults to current time; 1-tap "estimate from photo" or "pick recent". | Log with `confidence: "estimated"` flag so detector can down-weight. |
| Restaurant instead of home | "Swap planned meal" action: removes scheduled entry, opens restaurant search (generic Panama meals DB: sancocho, arroz con pollo, hojaldre, etc.). | Restaurant items stored with `source: "generic_estimate"`, wider error bars. |
| Snack discovered in kitchen | Persistent "Quick snack" shortcut → last-10 snacks list → 1-tap log. | Normal entry. |
| Forgot to log earlier meal | Allow backdated entry up to 48 h. Ask "approximate time?". Refuse >48 h to keep streaks honest. | Flag `backdated: true`; if >30% of entries are backdated, nudge user toward real-time logging. |
| Shared/partial portion | Slider "How much did you actually eat? 25/50/75/100%". | Multiply grams accordingly. |
| Not sure what it was | "Log 'unknown meal' with estimated kcal, review later" → adds a to-do in Insights tab. | Excluded from micronutrient detector until resolved. |

General principle: it's better to log an approximate meal than nothing.
Detector weights recent fully-specified entries more heavily, and explicitly
shows data quality ("Based on 5 of 7 logged days, 72% complete").
